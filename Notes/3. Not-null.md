### Constraints

- Constraints are rules applied to columns in a table.
- They maintain data accuracy and integrity.

#### A. NOT NULL

Prevents NULL values.

##### Example
```
CREATE TABLE students (
    id INT,
    name VARCHAR(50) NOT NULL
);
```
##### Meaning
- name must contain a value.
- Cannot be empty (NULL).


#### B. UNIQUE

- Prevents duplicate values.

##### Example
```
CREATE TABLE users (
    id INT,
    email VARCHAR(100) UNIQUE
);
```
##### Meaning
- Same email cannot appear twice.

#### C. DEFAULT

Sets a default value automatically.

##### Example
```
CREATE TABLE orders (
    order_id INT,
    status VARCHAR(20) DEFAULT 'Pending'
);
```
