# FOREIGN KEY?

- A FOREIGN KEY ensures data integrity.
- It prevents invalid references.

#### Example
```
CREATE TABLE orders (
    order_id INT,
    customer_id INT,
    FOREIGN KEY (customer_id)
    REFERENCES customers(id)
);
```
Now:

- customer_id in orders
- must exist in customers.id
- Without FOREIGN KEY

You could insert invalid data:

| order_id | customer_id |
| -------- | ----------- |
| 103      | 999         |



But customer 999 does not exist ❌





## FOREIGN KEY with CASCADE
#### 1. ON DELETE CASCADE

If parent row is deleted,
child rows are automatically deleted.

Example
### Parent Table
```
CREATE TABLE customers (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);
```
### Child Table
```
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    FOREIGN KEY (customer_id)
    REFERENCES customers(id)
    ON DELETE CASCADE
);
```

###### customers

| id | name |
| -- | ---- |
| 1  | Ravi |


###### orders

| order_id | customer_id |
| -------- | ----------- |
| 101      | 1           |
| 102      | 1           |


#### Delete Parent

```
DELETE FROM customers
WHERE id = 1;
```
