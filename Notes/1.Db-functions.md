### 🧮 1. COUNT() → count rows
👉 Total number of records
```
SELECT COUNT(*) FROM teacher;
```

With condition:

```
SELECT COUNT(*) 
FROM teacher 
WHERE salary > 25000;
```
### 💰 2. SUM() → total value

👉 Total salary of all teachers

```
SELECT SUM(salary) 
FROM teacher;
```


### 📊 3. AVG() → average value
👉 Average salary
```
SELECT AVG(salary) 
FROM teacher;
```


### 🔝 4. MAX() and MIN()
```
SELECT MAX(salary) FROM teacher;
SELECT MIN(salary) FROM teacher;
```
👉 Highest and lowest salary

### 👥 5. GROUP BY

- Used to group rows based on a column.
- Example: Group by specification

```
SELECT specification, COUNT(*) 
FROM teacher 
GROUP BY specification;

👉 Output:

.net → 2
java → 3
python → 1

```
Another example: average salary per department
```
SELECT specification, AVG(salary) 
FROM teacher 
GROUP BY specification;
```
⚠️ Rule to remember

- When using GROUP BY:
👉 Every column in SELECT must be:

- Either in GROUP BY
   Or inside an aggregate function (COUNT, SUM, etc.)

### 🔍 6. HAVING (after GROUP BY)

Used to filter grouped data (like WHERE but for groups)

Example:
```
SELECT specification, COUNT(*) 
FROM teacher 
GROUP BY specification
HAVING COUNT(*) > 1;
```
- Shows only specifications with more than 1 teacher

Another example:
```
SELECT specification, AVG(salary) 
FROM teacher 
GROUP BY specification
HAVING AVG(salary) > 25000;
```
