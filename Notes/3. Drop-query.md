# 💣 What DROP does

- It deletes the structure and the data permanently. 
- Once executed, the object is gone (unless you have a backup).

### 🧱 Common examples

#### 1. Drop a table
```
DROP TABLE employees;

➡️ Deletes the entire employees table and all its data.
```
#### 2. Drop a database
```
DROP DATABASE company_db;

➡️ Removes the whole database, including all tables inside it.
```
#### 3. Drop a column (in some databases)
```        
ALTER TABLE employees
DROP COLUMN email;
```
➡️ Removes only the email column from the table.

⚠️ Important warnings
```
Irreversible:       You cannot undo a DROP unless you restore from backup.
Deletes everything:  Both structure and data are removed.
Use with extreme caution in production environments.
```
#### 🔍 DROP vs DELETE vs TRUNCATE
```
DROP        → removes the whole table (structure + data)
DELETE      → removes specific rows (you can use WHERE)
TRUNCATE    → removes all rows but keeps the table
```
